==============================================================
  CSV EXTRACTOR — PRODUCTION DEPLOYMENT CHECKLIST
==============================================================

PREREQUISITES
  - A Render account (free tier works: https://render.com)
  - A Vercel account (free tier: https://vercel.com)
  - A PostgreSQL provider (Render built-in, Neon, or Supabase)
  - A Redis provider (Upstash free tier or Render Redis)
  - S3-compatible storage (AWS S3, Cloudflare R2, or MinIO)
  - At least one AI provider key (Groq, HF, or OpenAI)

ENVIRONMENT (.env)
  Copy .env.example → .env and fill in all values

╔══════════════════════════════════════════════════════════════╗
║                   RENDER — BACKEND DEPLOY                   ║
╠══════════════════════════════════════════════════════════════╣
║                                                              ║
║ 1. Push code to GitHub (or connect Render to your repo)      ║
║                                                              ║
║ 2. Create a new Web Service on Render:                       ║
║    - Name: csv-extractor-backend                              ║
║    - Root Directory: backend                                  ║
║    - Runtime: Node                                           ║
║    - Build Command: npm install && npx prisma generate        ║
║    - Start Command: npm start                                 ║
║    - Instance Type: Free                                      ║
║                                                              ║
║ 3. Add Environment Variables in Render dashboard:             ║
║    NODE_ENV=production                                        ║
║    PORT=3000                                                  ║
║    DATABASE_URL=postgresql://... (see step 4)                 ║
║    REDIS_URL=redis://... (see step 5)                         ║
║    MINIO_ENDPOINT=https://your-s3-endpoint                    ║
║    MINIO_ACCESS_KEY=...                                       ║
║    MINIO_SECRET_KEY=...                                       ║
║    MINIO_BUCKET=csv-uploads                                   ║
║    MINIO_REGION=us-east-1                                     ║
║    CORS_ORIGIN=https://your-frontend.vercel.app               ║
║    GROQ_API_KEY=gsk_...                                       ║
║    HF_TOKEN=hf_...                                            ║
║    OPENAI_API_KEY=sk-...                                      ║
║    JWT_SECRET=<random 64-char hex string>                     ║
║    BATCH_SIZE=500                                             ║
║                                                              ║
║ 4. PostgreSQL — Use Render's built-in Postgres:               ║
║    - Create a new Postgres DB in Render dashboard             ║
║    - Copy the Internal Database URL into DATABASE_URL         ║
║    - Run migrations after deploy (step 6)                     ║
║                                                              ║
║ 5. Redis — Use Upstash (free) or Render Redis:               ║
║    - Upstash: create a free Redis DB, copy REST URL           ║
║    - Render: create Redis service, copy Internal URL          ║
║                                                              ║
║ 6. After first deploy, run via Render Shell:                   ║
  ║    npx prisma migrate deploy                                  ║
  ║    npx prisma db seed                                         ║
  ║    (Creates tables and admin user: admin@test.com / admin123)  ║
  ║                                                              ║
  ║ 7. (Optional) Enable persistent disk for Render:              ║
  ║    Render free tier has ephemeral storage. If you need         ║
  ║    persistent disk for MinIO uploads, upgrade to a paid       ║
  ║    plan or use an external S3 provider (AWS S3, R2, etc).     ║
  ║    Set MINIO_ENDPOINT, MINIO_ACCESS_KEY, MINIO_SECRET_KEY     ║
  ║    to point at your external S3-compatible storage.           ║
  ║                                                              ║
  ║ 8. Verify:                                                     ║
  ║    curl https://your-backend.onrender.com/api/health           ║
  ║    Expected: {"status":"ok","database":"healthy"}              ║
  ║    curl -X POST <backend>/api/csv/sessions/<id>/ai-summary     ║
  ║    (generates AI dataset analysis — cached in Redis 5 min)     ║
║                                                              ║

SECURITY CHECKS
  √ .env not committed (in .gitignore)
  √ JWT authentication on all /api/csv/ routes
  √ RBAC permissions checked per route
  √ Multer fileFilter restricts to CSV only
  √ Input validation on registration (email, password length)
  √ Passwords hashed with bcrypt
  √ CORS restricted to frontend origin
  √ Rate limiting on auth routes

MONITORING
  - Backend logs to stdout (structured JSON via pino)
  - Render dashboard: view logs, restart, scale
  - Vercel dashboard: deployment logs, analytics
  - PostgreSQL: monitor connection pool size
  - Redis: monitor memory usage

SCALING
  - Backend: stateless → horizontal scale behind load balancer
  - Frontend: Vercel auto-scales globally
  - Database: upgrade Render Postgres plan or use Neon/Supabase
  - Redis: upgrade Upstash plan for more connections
  - Storage: use AWS S3 instead of MinIO for multi-region
  - BATCH_SIZE: increase for larger CSV imports
